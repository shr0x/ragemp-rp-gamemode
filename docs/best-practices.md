# Best Practices

---

## Naming

* Events: `system::action`
* Events: `system::action:performer`
* Files: `feature.module.ts` or `Feature.module.ts`

---

## Structure

* Keep files small
* Avoid deep nesting

---

## Logic

* Server = logic
* Client = behavior
* UI = display

---

## Consistency

Follow existing patterns.

---

## Golden Rule

> If your code feels messy — you are breaking the framework philosophy.
